## 1. 프로젝트 개요

- **주제**: 일상에서 발생하는 **수입·지출 내역을 기록·조회하고, 수입/지출/잔액을 자동 집계**하는 가계부 웹 서비스.
- **선정 이유**: 가계부는 누구나 매일 사용하는 데이터이며, "기록(C) → 목록(R) → 수정(U) → 삭제(D)"와 **합계 집계**가 자연스럽게 데이터베이스 활용을 요구한다. 교재 예제(연락처·블로그)와 주제가 겹치지 않는다.
- **구조**: 교재 `myBlog` 프로젝트 구조(express-ejs-layouts 레이아웃 + main/admin 라우트 분리 + 관리자 대시보드)를 기반으로 가계부 도메인에 맞게 구성했다.

## 2. 주요 기능

| 구분 | 기능 | 설명 |
|------|------|------|
| 인증 | 회원가입 / 로그인 / 로그아웃 | 비밀번호는 bcrypt로 해시 저장, 로그인 시 JWT를 발급해 httpOnly 쿠키로 보관 |
| 내역 | 추가 (Create) | 구분(수입/지출)·날짜·분류·금액·메모 입력 |
| 내역 | 목록 (Read) | 로그인한 사용자의 내역을 **최신 날짜순**으로 표시 |
| 내역 | 수정 (Update) | 기존 값이 채워진 폼에서 수정 (method-override로 PUT) |
| 내역 | 삭제 (Delete) | 행 단위 삭제 (method-override로 DELETE) |
| 집계 | 요약 | **수입 합계 / 지출 합계 / 잔액**을 상단 카드에 자동 계산 |
| 보안 | 접근 제어 | 미로그인 시 `/allTransactions` 등 접근을 `/admin`(로그인)으로 리다이렉트, 데이터는 사용자별로 격리 |

## 3. 기술 스택

- **런타임/서버**: Node.js, Express 5
- **뷰**: EJS + **express-ejs-layouts**(main / admin / admin-nologout 레이아웃)
- **데이터베이스**: MongoDB + Mongoose (ODM)
- **인증**: bcrypt(비밀번호 해시), jsonwebtoken(JWT), cookie-parser
- **기타**: method-override(폼에서 PUT/DELETE), dotenv(환경변수), express-async-handler(비동기 에러 처리)
- **배포**: GitHub → Render(웹 호스팅) → MongoDB Atlas(클라우드 DB)

## 4. 데이터 모델

**User** — 사용자 계정
| 필드 | 타입 | 제약 |
|------|------|------|
| username | String | 필수, 유일(unique) |
| password | String | 필수(bcrypt 해시) |

**Transaction** — 가계부 내역
| 필드 | 타입 | 제약 |
|------|------|------|
| user | ObjectId(ref: User) | 필수 — 소유자 |
| type | String | 필수, `income`/`expense` |
| category | String | 필수 — 분류(식비, 교통, 급여 …) |
| amount | Number | 필수, 0 이상 |
| memo | String | 선택 |
| date | Date | 기본값 현재 시각 |
| createdAt/updatedAt | Date | timestamps 자동 생성 |

> `user` 필드로 사용자와 내역을 1:N으로 연결하고, 모든 조회·수정·삭제 쿼리에 `user: 로그인ID` 조건을 걸어 데이터를 격리한다.

## 5. 아키텍처 / 폴더 구조

`myBlog` 구조를 따른다. 컨트롤러를 따로 두지 않고 **라우트 파일에 로직을 작성**하며, 화면은 **레이아웃(express-ejs-layouts)** 위에 각 뷰의 `body`가 끼워지는 방식이다.

```
myLedger/
├─ app.js                 # 진입점: 레이아웃/미들웨어/라우터 등록, 서버 기동
├─ config/db.js           # MongoDB 연결 (connectDb)
├─ models/                # User.js, Transaction.js (Mongoose 스키마)
├─ routes/
│   ├─ main.js            # 공개 페이지: / (홈), /about
│   └─ admin.js           # 로그인/회원가입/로그아웃 + 가계부 CRUD + 집계 (checkLogin 포함)
├─ views/
│   ├─ index.ejs, about.ejs
│   ├─ admin/             # index(로그인·가입), allTransactions(목록+집계), add, edit
│   └─ layouts/           # main, admin, admin-nologout
├─ public/css/style.css   # 정적 자원
├─ .env                   # 환경변수 (DB_CONNECT, JWT_SECRET) — git 제외
└─ .gitignore             # .env, node_modules 제외
```

**요청 흐름 (예: 내역 추가)**
`브라우저 POST /add` → `routes/admin.js` → `checkLogin`(쿠키의 JWT 검증, `req.userId` 주입) → `Transaction.create`(Mongoose 저장) → `/allTransactions`로 리다이렉트 → 목록 + 합계 렌더

## 6. 로컬 실행 방법

```bash
npm install
# .env 작성
#   DB_CONNECT = mongodb://127.0.0.1:27017/myLedger   (또는 Atlas 주소)
#   JWT_SECRET = 임의의_비밀키
npm start            # = node app.js  → http://localhost:3000
```

## 7. 배포 방법 (GitHub + MongoDB Atlas + Render)

1. **MongoDB Atlas**: 무료 클러스터 생성 → Database Access에서 DB 사용자(ID/PW) 생성 → Network Access에서 `0.0.0.0/0` 허용 → Connect ▸ Drivers에서 `mongodb+srv://...` 연결 주소 복사.
2. **GitHub**: 프로젝트를 새 저장소에 push. `.env`와 `node_modules`는 `.gitignore`로 제외된다.
   ```bash
   git init && git add . && git commit -m "Init my ledger"
   git branch -M main
   git remote add origin <내-저장소-주소>
   git push -u origin main
   ```
3. **Render**: New ▸ Web Service → GitHub 저장소 연결 → 설정
   - Runtime `Node` / Build `npm install` / Start `node app.js` / Instance `Free`
   - **Environment Variables** (가장 중요):
     - `DB_CONNECT` = Atlas의 `mongodb+srv://...` 주소 (끝에 DB명 `/myLedger` 권장)
     - `JWT_SECRET` = 임의의 비밀키
   - Create Web Service → 로그에 `Your service is live!` 가 뜨면 `https://<프로젝트명>.onrender.com` 주소로 접속.

> 코드의 포트는 `const port = process.env.PORT || 3000` 으로 Render가 주는 포트를 따르도록 작성되어 있다.

## 8. 채점 기준 대응

- **안정적 배포**: Render에서 `npm install` → `node app.js`로 기동, `process.env.PORT` 사용, 민감정보는 환경변수로 분리.
- **데이터베이스 활용**: MongoDB Atlas에 사용자/거래 데이터를 저장하고, 조회·정렬·집계(수입/지출/잔액)·수정·삭제를 모두 DB 기반으로 수행한다.
